Section 02 — Protocols

Published protocols

Each product runs a named, versioned protocol. This section documents what each protocol trains, how it makes decisions, what data it logs, and where its limits are.

2 protocols · Jan 2025
⚡
Zone Coach
Readiness detection + training gate routing
v2.1 Updated Jan 2025 ↗ Full spec
What it trains
Cognitive readiness awareness. Pre-session state detection using the CCC (Cognitive Control Capacity) battery. Trains the user to recognise and act on their cognitive state before training begins.
Checks used
CCC battery Reaction time Interference sensitivity Attention consistency
3-minute check producing one of three outputs: Flat, In the Zone, or Spun Out.
Data logged
Session timestamp, CCC output state, reaction time distribution, consistency score. Stored locally only. No data leaves the device unless the user exports via JSON. No account required.
Routing logic
State output gates training recommendations: Flat → recovery block, Zone → challenge block, Spun Out → stabilise first. Routing is deterministic and fully described in the full protocol spec.
⚠ Limits
CCC output is an indicator, not a clinical assessment. A single check is a signal, not a diagnosis. State can change during a session. The protocol does not account for medical conditions, medication effects, or sleep disorders.
🧠
Capacity Gym — Trident G Far Transfer Protocol
Adaptive n-back · relational processing · thin automation reduction
v1.4 Updated Dec 2024 ↗ Full spec
What it trains
Working memory, attention control, and relational processing. Specifically designed to reduce thin automation — where performance gains do not generalise beyond the exact training format.
Training modes
Hub n-back (3-modality) Relational: Transitive Relational: Graph Relational: Propositional Swap probes Quiz probes
Coaching cycle
Four-phase cycle per session block: Stabilise → Perturb → Select → Consolidate. One dial adjusted at a time.
Data logged
Block-level accuracy, reaction time, n-back level, wrapper type, probe results. Stored locally. JSON export. No external transmission.
⚠ Limits
Far transfer is probabilistic, not guaranteed. Relational n-back modes are harder than standard n-back but do not constitute a clinical intervention. Performance on in-app tasks does not directly predict real-world outcomes.
