# Lane Operating Model

## Revenue Split Rule (example implementation)

- Base partnership split: `60/40` on distributable net sales.
- Applied per lane, per month, based on active contribution threshold.

## Active Contribution Definition

A contributor is active for a lane in a month when at least one of the below is completed and logged:

1. Market validation outputs:
   - interviews
   - qualification notes
   - lane demand summary
2. Offer and positioning outputs:
   - lane-specific offer spec
   - copy changes adopted into shipped assets
3. Product and UX outputs:
   - interface direction accepted into release scope
   - lane-specific onboarding or recommendation logic inputs
4. Delivery operations outputs:
   - booking/support/coaching lane process updates
   - partner execution artifacts
5. Sales execution support:
   - proposal support
   - lane-specific close support
   - documented partner-introduced opportunities

## Evidence Standard

- Contribution entries must be written to a monthly ledger with:
  - date
  - lane
  - output type
  - artifact URL
  - acceptance status
- Disputes resolve against this ledger, not memory or verbal summaries.

## Lane KPIs (minimum)

1. Qualified leads
2. Conversion to paid
3. Activation completion
4. 2-week retention
5. Recovery/stability metrics (where protocol data exists)
