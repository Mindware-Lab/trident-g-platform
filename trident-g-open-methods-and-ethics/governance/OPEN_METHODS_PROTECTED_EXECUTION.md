# OPEN_METHODS_PROTECTED_EXECUTION.md

We build in public where transparency increases trust, auditability, and scientific value - without treating \"open\" as \"anything goes\".

## Open methods layer

Published openly:
- Protocol goals and boundaries
- Constructs and progression logic families
- Validation logic families and caveat labels
- Metric definitions, scoring categories, and schema contracts
- Claims boundaries, ethics posture, and privacy controls
- Reproducible examples with synthetic/non-identifiable sample data

## Protected execution layer

Kept private:
- Runtime app/source code and private infrastructure
- Anti-abuse and anti-gaming heuristics
- Private operational thresholds and sequencing constants
- Payment/backend operations and access controls
- Internal onboarding/coaching/clinic playbooks
- Premium dashboards, reports, and support workflows

## Data posture: privacy by default

Default:
- Local-first operation where available
- No account required for core method inspection

Optional sharing modes (when offered by products):
1. Local-only (default)
2. Opt-in anonymized summaries
3. Research mode with separate consent and withdrawal process

We do not accept identifiable clinical data via public channels.

This boundary supports scientific inspectability without exposing exploit paths.
