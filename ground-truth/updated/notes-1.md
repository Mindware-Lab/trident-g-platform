## 1. Structural Overview

The **G loop** is not a simple sequential process. It is a **concurrent multi-layer dynamical system** in which specific transition events mark qualitative regime changes.

Four layers operate simultaneously, coupled through:

- shared **α stability monitoring**
- the **hippocampal successor representation (SR) map system**
- fast and slow **Gc construction pathways**
- allostatic regulation through the **Ψ-band**

Each loop iteration is a **spiral rather than a cycle**. It returns to a representationally richer starting state through the slow Gc gist extraction pathway, which installs cross-loop invariants into cortical representational geometry.

# Part I — The Four-Layer G Loop Architecture

---

## 2. Layer 1: The Sustained Bifurcating Dynamic  
**Always running — constitutes the Griffiths Phase regime itself**

Layer 1 is the core dynamical engine. It maintains the system in an extended near-critical regime through the competition between entropy-driven exploration and mutual-information-driven constraint.

---

### 2.1 FPCN-B Entropy Arm Activation

The FPCN-B entropy arm supports broad associative search across relational map space via the hippocampal intermediate-horizon SR system.

This is **not random exploration**. It is **SR-structured search**:

- successor representations are weighted by temporal discounting
- candidate relational configurations are generated
- these configurations are neither purely local nor purely global
- the default mode network contributes autobiographical and semantic depth

This corresponds to the **Zhang–Tang entropy maximisation process**: the system maintains maximal flexibility across the current state of knowledge.

The entropy arm’s reach depends on the richness of the intermediate-horizon SR maps installed by prior loop iterations through the slow Gc pathway.

---

### 2.2 FPCN-A Mutual Information Constraint

The FPCN-A mutual-information arm applies simultaneous goal-directed constraint to the relational configurations generated by the entropy arm.

Its functions include:

- weighting candidate configurations by task relevance
- using prediction-error signals from real-world outcomes
- maintaining the current objective function in working memory
- constraining search without prematurely closing it

This corresponds to the **Zhang–Tang mutual information constraint**: updates are filtered for task-relevant dependency between representations and outcomes.

The precision of this constraint arm depends on the quality of fast Gc compilations available as structured building blocks from prior loop iterations.

---

### 2.3 Griffiths Phase Regulation

The maintained competition between FPCN-B entropy and FPCN-A mutual information prevents the system from collapsing into either:

- pure exploitation
- pure exploration

This balance is monitored through **α stability**: whether the system’s representational update distribution maintains heavy-tailed, power-law statistics consistent with Griffiths Phase operation.

#### Regulation rules

When **mutual information saturation** is detected:

- the update distribution flattens toward thin tails
- the system risks subcritical collapse
- entropy injection is triggered

When **entropy excess** is detected:

- update distributions become extremely heavy-tailed
- convergence fails
- mutual-information tightening occurs

This regulation keeps the system within the Griffiths Phase regime rather than drifting into subcritical rigidity or supercritical fragmentation.

The GP regulation function is the computational analogue of regional heterogeneity and structural modularity producing an extended critical regime. It prevents any single module’s bifurcation resolution from dominating the global dynamic.

## 3. Layer 2: The SR Tuning Dynamic  
**Operating within the GP regime on the currently active relational map**

Layer 2 describes how the active SR map is tuned, extended, and coupled across timescales.

---

### 3.1 Horizontal SR Tuning

Within the currently active relational map, the system tunes its successor representation across surface variation.

Examples of variation include:

- different wrappers
- different feature instantiations
- different contextual embeddings

This is the interior of a power-law learning segment: efficient exploitation of the current relational structure generating smooth performance gains.

The key variable is the **SR temporal horizon**.

Horizontal tuning deepens the map’s predictive accuracy at the current horizon without yet extending it.

#### Characteristic signature

- monotonically improving performance
- decreasing update-magnitude variance
- progressive mutual-information constraint
- no full saturation yet

---

### 3.2 Temporal Horizon Probing

Simultaneously, the system probes whether extending the SR temporal horizon reveals deeper relational structure.

This means asking whether the current map’s predictions improve if the discount factor is relaxed to include more distal state transitions.

This is not deliberate strategy. It is an automatic property of the hippocampal SR system.

When longer-horizon predictions add explanatory value:

- the active SR map extends its horizon
- deeper structural features become visible
- a bifurcation response may begin before performance fully plateaus

This probe allows the loop to self-initiate a bifurcation response before the asymptote is completely reached.

---

### 3.3 Cross-Scale Coupling

Three timescales are coupled rather than sequential:

| Timescale | Function |
|---|---|
| Fast subcortical SR automations | Provide compiled Gc building blocks |
| Intermediate hippocampal SR maps | Recombine structured relational material |
| Slow prefrontal schemas | Provide boundary conditions and strategic constraints |

The coupling runs in both directions:

1. subcortical automations feed upward into hippocampal SR maps  
2. hippocampal SR maps feed upward into prefrontal schemas  
3. prefrontal schemas feed downward to reshape what the hippocampal SR treats as relevant  
4. this reshapes which subcortical automations are recruited  

The richness of this cross-scale coupling determines the maximum complexity of relational structures available to the current loop iteration.

- sparse coupling → shallow relational maps  
- rich coupling → deep relational maps

## 4. Layer 3: The Active Gc Construction Dynamic  
**Operating continuously on two distinct pathways**

Layer 3 explains how Gf activity becomes Gc structure through fast compilation and slow gist extraction.

---

### 4.1 Prediction-Error Gc Pathway — Fast Compilation

When the FPCN-A constraint becomes sufficiently strong, the bifurcating dynamic resolves.

This occurs when a specific relational configuration has been validated across surface variation so that uncertainty about its task relevance is genuinely low.

At this point, the configuration crosses the bifurcation threshold into **subcritical SR automation**.

This is **fast Gc banking**.

#### Features of fast Gc banking

- specific compiled strategies
- documented boundary conditions
- portability signatures
- timescale: deliberate practice, sessions to weeks
- model-free and subcortical-dominant
- post-bifurcation

Once compiled, the structure no longer internally bifurcates, but it remains available to FPCN-B as a structured building block for novel recombination.

---

### 4.2 Self-Organising Gc Pathway — Slow Gist Extraction

The slow pathway operates continuously below the threshold of deliberate attention.

The hippocampal system extracts statistical regularities recurring across:

- episodes
- fast Gc compilations
- domains
- contexts
- loop iterations

This gist is progressively installed in cortical representational geometry through consolidation, especially during sleep and rest.

#### Output

The output is not a compiled strategy. It is a **schematic geometry**: a restructured representational landscape in which certain relational invariants become primary dimensions.

This makes future entropy-arm search more likely to land on deep portable solutions rather than surface recombinations.

#### Timescale

- sleep consolidation
- rest intervals
- extended practice
- weeks to months

---

### 4.3 Consolidation Interval Dependency

The slow Gc pathway requires genuine consolidation intervals: periods without new prediction-error override.

Without adequate consolidation:

- episodic material accumulates
- schematic geometry is not installed
- the loop loses its spiral property
- each iteration returns to approximately the same representational starting state

In this case, the loop degrades into flat cycling rather than self-enriching spiralling.

Adequate consolidation is therefore not a rest period in a weak sense. It is an active architectural requirement for long-term intelligence development.

## 5. Layer 4: The Individual Differences Layer  
**Characteristic loop signatures across stable FPCN-A/B balance profiles**

The individual differences layer characterises how stable biases in FPCN-A/B balance produce distinctive patterns across the loop’s phases and transition events.

Rather than treating individual differences as fixed capacity variations, this layer treats them as **characteristic dynamical signatures**.

These signatures appear in:

- α stability maintenance
- transition event timing
- Gc pathway activation
- where the loop is most constrained

Five archetypal profiles are identified. Real individuals are blends of these profiles, and profiles can shift across domains and developmental periods.

---

## 6. Profile 1: Balanced GP  
**High Gf, integrated Gc/Gf**

---

### 6.1 Dynamical Signature

α stability is maintained across all loop phases without characteristic collapse points.

Transition events occur at approximately theoretically optimal timing:

- asymptote detection before complete plateau
- bifurcation response generates genuine higher-order structure
- portability validation cascade completes before fast Gc banking

Both Gc pathways are active and well-coupled:

- fast Gc compilations are genuinely portable
- slow Gc gist extraction reshapes subsequent search spaces
- consolidation intervals are used effectively

---

### 6.2 Layer 1 Signature

FPCN-A/B balance is dynamically modulated by task demands.

- entropy arm leads on novel configurations
- mutual-information arm leads on validation and constraint
- neither dominates long enough to collapse the bifurcating dynamic

The product `I·β` remains approximately stable across loop phases, indicating genuine self-organised criticality rather than externally imposed balance.

---

### 6.3 Layer 2 Signature

Horizontal SR tuning is efficient but not premature.

Temporal horizon probing is active and productive. Horizon extensions frequently reveal genuine structural depth, creating self-initiated bifurcation responses before full performance plateau.

Cross-scale coupling is rich:

- subcortical automations
- hippocampal SR maps
- prefrontal schemas

are all mutually coupled and mutually reshaping.

---

### 6.4 Layer 3 Signature

Fast Gc compilations are genuinely cross-domain portable.

They survive:

- wrapper variation
- boundary testing
- novel context deployment

Slow Gc gist extraction is productive, installing cross-loop invariants as schematic geometry.

---

### 6.5 Training Implication

This profile benefits most from increasing the complexity and novelty of relational structures.

Appropriate challenge level:

- small-world composite problems
- applied scenario training
- top of the vertical stack

Primary risk:

- complacency
- insufficient novelty
- over-consolidation by the mutual-information arm without bifurcation response

## 7. Profile 2: Mutual Information Dominant  
**High Gc, constrained Gf**

---

### 7.1 Dynamical Signature

α stability collapses early in the bifurcation response phase.

The update distribution flattens toward thin tails before genuine higher-order structure is found.

Transition Event 2 is truncated:

- entropy arm activates
- mutual-information arm recaptures the dynamic too early
- existing Gc schema constraints are applied prematurely
- fast Gc banking occurs before full portability validation

This produces wrapper-specific automations that appear robust in familiar contexts but fail under genuine surface variation.

---

### 7.2 Layer 1 Signature

FPCN-A activates strongly and quickly on novel problems.

The mutual-information arm applies existing constraint schemas before FPCN-B has completed its search.

The `I·β` product is unstable toward mutual-information dominance:

- λ rises quickly
- β collapses
- α falls below the optimal heavy-tailed range

The system becomes subcritical during what should be the GP bifurcation response phase.

---

### 7.3 Layer 2 Signature

Horizontal SR tuning is highly efficient within familiar relational structures.

This profile often shows steep within-segment power-law performance curves.

However:

- temporal horizon probing is limited
- horizon extension is blocked by existing schema constraints
- cross-scale coupling is weighted toward subcortical-to-hippocampal direction
- existing automations dominate recombination space

---

### 7.4 Layer 3 Signature

Fast Gc compilations accumulate rapidly but lack genuine portability.

Boundary conditions are underspecified and wrapper sensitivity is high.

Slow Gc gist extraction operates on a narrow episodic base, producing refined domain knowledge rather than genuinely cross-domain invariants.

---

### 7.5 Training Implication

This profile requires entropy injection at the bifurcation response phase.

Useful training features:

- tasks that resist existing FPCN-A constraint schemas
- non-verbal geometric n-back
- temporal relation n-back
- explicit wrapper variation before consolidation
- boundary case exposure before fast banking

The Donner-Hardy breakpoint should be deliberately extended in training so that the person repeatedly encounters the plateau before consolidation is permitted.

The aim is to build tolerance for the bifurcation response phase rather than permitting premature escape into fast Gc banking.

## 8. Profile 3: Entropy Dominant  
**High creativity, constrained consolidation**

---

### 8.1 Dynamical Signature

α stability collapses in the opposite direction.

The update distribution develops extreme heavy tails without convergence.

Transition Event 3 is delayed or fails:

- many candidate relational configurations are generated
- mutual-information constraint is too weak to select among them
- fast Gc banking is rare
- the bifurcation threshold is rarely crossed

Slow Gc gist extraction is active but poorly anchored because there are insufficient validated fast Gc structures.

---

### 8.2 Layer 1 Signature

FPCN-B activates strongly and broadly on novel problems.

The entropy arm generates rich, far-reaching associative search.

However, FPCN-A applies insufficient constraint:

- mutual-information engagement is intermittent or weak
- `I·β` is unstable toward entropy dominance
- β remains high
- λ fails to rise sufficiently
- α exceeds the optimal heavy-tailed range

This creates supercritical rather than GP-regime dynamics.

---

### 8.3 Layer 2 Signature

Horizontal SR tuning is inefficient because entropy dominance prevents stable constraint.

Temporal horizon probing is highly active but undirected.

Horizon extensions are frequent but not filtered for task relevance, producing a broad but shallow SR map.

Cross-scale coupling is weighted toward hippocampal-to-prefrontal flow:

- diverse episodic material accumulates
- stable subcortical automations are not formed reliably

---

### 8.4 Layer 3 Signature

Fast Gc compilations are rare and often unstable.

Boundary conditions are poorly specified, and performance under pressure is fragile.

Slow Gc gist extraction is rich in episodic diversity but produces schematic geometry that is creative yet unreliable.

---

### 8.5 Training Implication

This profile requires mutual-information constraint training.

Critical training priorities:

- FPCN-A goal-relevant constraint
- MFT-M cognitive control at the base of the vertical stack
- structured portability validation cascades
- clear pass/fail criteria
- consolidation discipline

This profile tends to resist consolidation by continuing to explore rather than allowing the slow Gc pathway to install gist as stable schematic geometry.

## 10. Profile 5: Consolidation Deficient  
**Active GP, degraded spiral**

### 10.1 Dynamical Signature

Layer 1 and Layer 2 dynamics are approximately Profile 1 quality:

The loop cycles rather than spirals.

GP dynamics are well maintained within each loop iteration.

The entropy arm’s search space at the start of each new iteration is not demonstrably richer than before, indicating slow Gc pathway failure.

- boundary conditions remain constant across loop iterations
- search spaces do not progressively refine

### 10.4 Layer 3 Signature

The slow Gc pathway fails because of one or more of the following:

- insufficient cross-domain episodic diversity
- impaired cortical consolidation
- narrow training material

The characteristic marker is a library of portable strategies without coherent higher-level schematic integration.

This is expertise without wisdom.

Genuine rest periods without prediction-error override are needed so the slow Gc pathway can complete gist extraction.

The slow Gc extractor needs material from genuinely different domains and contexts to find cross-loop invariants.

Useful training:

- small-world composite tasks
- applied scenario training
- top-of-stack integration work
- sleep hygiene
- stress management

Sleep and recovery are not peripheral concerns. They are direct interventions on the slow Gc consolidation mechanism.

## 11. Overview

Transition events mark qualitative regime changes across all layers simultaneously.

---

## 12. Transition Event 1: Asymptote Detection

The current SR map’s performance gains flatten. The power-law learning segment reaches its local ceiling.

This is detected by the GP regulation subsystem as mutual-information saturation:

- prediction-error signal no longer generates sufficient update magnitude
- the update distribution flattens toward thin tails
- the system approaches subcriticality within the current map

### Individual differences marker

| Profile | Characteristic pattern |
|---|---|
| Profile 2 | Detects asymptote late, after subcritical collapse |
| Profile 3 | Continues exploring past the plateau |
| Profile 1 | Detects near optimal timing |
| Profile 4 | Detects near optimal timing, but at shallow depth |

---

## 13. Transition Event 2: GP Bifurcation Response

The FPCN-B entropy arm reactivates at increased intensity.

The hippocampal SR temporal horizon extends beyond its current tuned value.

The default mode network’s associative reach expands into domains not previously connected to the current relational structure.

FPCN-A mutual-information constraint relaxes. It does not disappear, but loosens enough to allow novel relational configurations to be evaluated.

### α stability signature

- heavy tails extend
- the system moves away from local asymptote
- relational search territory opens

### Profile-specific failures

| Profile | Failure mode |
|---|---|
| Profile 2 | FPCN-A recaptures the dynamic too early |
| Profile 3 | The event continues indefinitely without convergence |

---

## 14. Transition Event 3: Higher-Order Structure Emergence

A qualitatively better relational configuration becomes available through entropy-arm search.

The new structure:

- accounts for the current map’s successes
- explains its failures as boundary cases
- reveals a more general invariant

In Zhang–Tang terms, this appears as:

- a large update event
- one of the power-law distributed jumps
- rapid mutual-information gain afterwards

Profile 4’s limitation occurs here: the new structure is genuine but shallow because SR depth constrains invariant complexity.

---

## 15. Transition Event 4: Portability Validation Cascade

The new structure is deployed across:

- wrapper variations
- boundary cases
- genuinely novel contexts

The GP regulation subsystem monitors whether α stability is maintained across these variations.

If α stability is maintained, the structure has genuine cross-domain portability.

If α stability collapses, the structure is a local solution masquerading as a general one, and Transition Event 2 is re-triggered.

### Profile-specific failures

| Profile | Failure mode |
|---|---|
| Profile 2 | Premature fast Gc banking bypasses validation |
| Profile 3 | Rarely reaches this event |
| Profile 5 | Validation works, but output does not feed into slow Gc |

---

## 16. Transition Event 5: Bifurcation Threshold Crossing

The new structure crosses the bifurcation threshold.

Mutual-information dominance resolves the bifurcating dynamic for this specific structure, compiling it into fast Gc as an SR automation with documented boundary conditions.

Simultaneously, the structure enters the slow Gc gist extraction pathway.

The next loop iteration begins from a representationally richer state.

This is where the spiral advances.

Profile 5’s failure occurs here: fast Gc compilation completes, but slow Gc installation fails before the next loop iteration begins.
